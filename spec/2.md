原因分析（问题13）：

根因1：useCommonEventModal 的 reset() 用的是冻结的默认值

- useCommonEventModal.ts 中，CREATE 打开时调用 reset()（无参），RHF 的 reset() 无参版本用的是 useForm 首次挂载时传入的 defaultValues，之后不更新
- EventOrderDrawer 是常驻挂载的（open 只控制显示），首次挂载时 selectedEvents 为空，eventInfo = undefined，所以 reset() 写入的是 attendanceEstimate: 0 或某个旧值

根因2：selectedEvents[0] 是旧引用

- useEventHandles.ts 的 onRowValueChanged 中，validateEvent(event) 返回新对象，applyTransaction({ update: [validatedEvent] }) 把 node.data 替换成新对象
- onSelectionChanged 存的是 node.data 的引用，grid 编辑后 node.data 被替换成全新的 validatedEvent 对象，但 store 里的 selectedEvents[0] 仍指向旧对象，attendance 还是改之前的值

修复方案（src/hooks/events/useEventOrderModal.tsx）：

1. 绕过根因2 —— 不走 selectedEvents，直接从 grid API 读最新数据：
   const selectedNodes = gridRef.current?.api.getSelectedNodes() || [];
   const freshEventInfo = selectedNodes[0]?.data;
   getSelectedNodes() 返回当前 grid 里的 node.data，即 applyTransaction 替换后的 validatedEvent，attendance 是最新值

2. 绕过根因1 —— 不依赖 reset() 的冻结默认值，显式传参：
   reset(getCreateDefaultValues(freshEventInfo));
   RHF 的 reset(values) 用的是传入的 values，不是冻结的 defaultValues

3. 执行顺序保证 —— React effects 按声明顺序执行：
   - line 136: useCommonEventModal(...) 内部 useEffect 先注册先执行，调用 reset()（写入旧值）
   - line 149: 新增的 useEffect 后注册后执行，调用 reset(getCreateDefaultValues(freshEventInfo))（用 grid 最新数据覆盖）
     最终表单值以后者为准

4. 提取 getCreateDefaultValues 函数，useForm 和 effect 共用，所有依赖 eventInfo 的字段都自动覆盖到
